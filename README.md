# 911 Call Data Analysis – Montgomery County, PA

## Project Overview
This project analyzes over 663,000 emergency 911 calls from Montgomery County, Pennsylvania, using SQL and database design principles. The goal is to uncover demand patterns and provide insights for staffing, resource allocation, and emergency response planning.

## Business Problem
Emergency services operate with limited personnel and budgets while facing fluctuating demand across time, location, and incident types. Without data-driven insights, planning often relies on assumptions, which can impact response times and efficiency.

## Dataset
- 663,522 emergency call records
- Time period: December 2015 – July 2020
- Categories: EMS, Traffic, Fire
- Source: Public 911 Call Dataset (Montgomery County, PA)

## Data Modeling
- Designed **Conceptual, Logical, and Physical data models**
- Implemented normalized tables with **primary keys and foreign keys**
- Used **InnoDB** to enforce referential integrity

## Analysis Highlights
- EMS accounts for ~50% of all emergency calls
- Call volume peaks during late afternoon hours (around 5 PM)
- Weekdays experience higher call volumes than weekends
- A small set of townships and intersections generate a significant share of total calls

## Tools & Technologies
SQL (MySQL), Database Design, ETL Concepts, R (for visualizations)

## Outcome
The analysis provides actionable insights to support smarter staffing, improved response readiness, and long-term emergency planning.
